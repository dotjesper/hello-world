## Getting started with GitHub, Git and Visual Studio Code

**Level:** 300+ | **Duration:** 45 minutes - 2+ hours | **Format:** Demo-heavy, hands-on walkthrough
**Audience:** Device administrators, IT professionals, and IT admins with no prior GitHub or Git experience

### Abstract

You already write code - you just don't call it that. Every PowerShell script, every configuration baseline, every deployment fix shared over email is code without a home. GitHub, Git, and Visual Studio Code give that code a home - with change history, safe collaboration, and a professional way to share your work.

As IT operations increasingly overlap with DevOps practices, version control is no longer optional - it is a core skill. Whether you manage device configurations through Microsoft Intune, maintain shared PowerShell scripts, or document processes for your team, the ability to track changes, collaborate safely, and share your work professionally sets you apart. This demo-heavy session is designed to accelerate your journey from first commit to confident daily use.

Based on personal experience and real-world lessons - many of which are collected at [dotjesper.com](https://dotjesper.com/) - this session covers the full journey from first install to daily use, with every step demonstrated live.

This is not a session for seasoned developers. It is designed for IT professionals who want to bring structure, collaboration, and modern workflows into their daily work.

Every concept is demonstrated live using the [Hello World](https://github.com/dotjesper/hello-world/) repository and its companion [wiki](https://github.com/dotjesper/hello-world/wiki), both publicly available for you to explore at your own pace. The repository and wiki serve as a take-home reference, covering everything shown in the session and more.

### What to expect

The session covers the following topics through live demonstrations:

- **Getting started** - Install Visual Studio Code and Git, configure your identity, and sign in with GitHub
- **Creating a repository** - Create and configure a GitHub repository with a README, license, wiki, and the configuration files that turn a folder into a professional project
- **Writing content** - Write documentation in Markdown, structure common repository files (README, CONTRIBUTING, .gitignore, .editorconfig, .gitattributes), and craft good commit messages
- **Branching and workflows** - Clone repositories, create branches, submit pull requests, set up multi-root workspaces, and practice the daily Git pull-stage-commit-push cycle
- **Working with repositories** - Explore GitHub URL patterns, download files and repositories programmatically using PowerShell and the GitHub REST API
- **AI as a learning companion** - Use GitHub Copilot to generate, review, and understand PowerShell scripts - write effective prompts, validate AI output, and discover new concepts like bitmasks
- **Field notes** - Walk through practical lessons learned through hands-on experience, including multi-root workspace quirks, wiki branch limitations, and GitHub Copilot custom instructions

### What you take home

Three key takeaways and a practical reference you can use immediately:

1. **GitHub is for IT professionals, not just developers.** Tracking PowerShell scripts, configuration files, and documentation in a version-controlled repository brings the same benefits to device management that it brings to software development - change history, safe collaboration, and professional sharing.

2. **A well-structured repository is professional branding.** Every public repository reflects how you work. Investing in clear documentation, consistent formatting, and proper configuration signals quality and makes your work accessible to others.
Yes pleas
3. **AI-assisted coding is a learning accelerator, not a shortcut.** GitHub Copilot helps you write, review, and understand code faster - but only when you engage critically with its output. The difference between a superpower and a crutch is whether you build understanding along the way.

Beyond the takeaways, participants leave with:

- An understanding of how to structure, configure, and maintain a professional GitHub repository - demonstrated through the [Hello World](https://github.com/dotjesper/hello-world/) repository and its companion [wiki](https://github.com/dotjesper/hello-world/wiki)
- The skills to apply the daily Git workflow - pull, stage, commit, push - to real-world IT tasks
- Confidence to start using GitHub Copilot as a learning companion for exploring new concepts and validating scripts
- Practical experience with PowerShell and the GitHub REST API for downloading files and working with repositories programmatically
